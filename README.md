# Distributed Key-Value Store using Raft Consensus

A production-inspired distributed key-value store built with Python and FastAPI, implementing core concepts of the Raft Consensus Algorithm.

## Features

* Leader Election
* Heartbeat Mechanism
* Log Replication
* Commit Index Tracking
* Persistent State Recovery
* Snapshotting and Log Compaction
* Snapshot Recovery
* Follower Catch-Up Synchronization
* Snapshot-Based Node Rejoin Recovery
* Cluster Health Monitoring
* Real-Time Dashboard
* Live Event Streaming
* Animated Cluster Visualization

## Technologies Used

* Python
* FastAPI
* HTML
* CSS
* JavaScript
* Git
* Raft Consensus Algorithm

## Architecture

Leader Node
↓
Heartbeats
↓
Follower Nodes

Leader Election
↓
Log Replication
↓
Commit
↓
Snapshot
↓
Recovery

## Project Highlights

* Simulates a distributed cluster with multiple nodes.
* Supports automatic leader failover.
* Maintains consistency using replicated logs.
* Provides recovery after node crashes.
* Includes a real-time monitoring dashboard.
